# X API 申请材料

## 项目信息

### 项目名称
openclaw-x-video

### 项目类型
自动化内容创作工具

### 项目描述
这是一个基于AI的自动化视频生产系统，用于将X平台上的热门内容转化为适合短视频平台（如抖音）的竖屏视频。系统监控X上的高互动内容，通过AI分析内容价值，自动生成文字动画或混剪视频。

### 主要功能
1. **内容发现**：每小时扫描X时间线，识别高互动推文
2. **AI分析**：使用LLM判断内容类型和爆款潜力
3. **视频生成**：自动渲染1080×1920竖屏视频
4. **队列管理**：串行处理确保服务器资源合理使用

### 技术栈
- Python + Playwright（备用方案）
- OpenAI-compatible LLM（DeepSeek）
- Remotion（React视频渲染）
- FFmpeg（视频处理）

## API使用计划

### 数据需求
- **频率**：每小时1次
- **数量**：每次20-50条推文
- **内容**：公开推文，按点赞数排序
- **字段**：推文文本、作者、互动数据（点赞、转发、回复）

### 使用场景
1. **趋势发现**：识别X上的热门话题
2. **内容分析**：AI判断是否适合视频转化
3. **创作辅助**：为视频文案提供素材

### 数据处理原则
1. **不存储用户数据**：只分析推文内容，不存储用户个人信息
2. **内容转换**：推文内容会被重新创作，不直接复制
3. **遵守规则**：严格遵守X开发者协议和内容政策
4. **频率限制**：合理控制API调用频率

## 申请要点

### 为什么需要API访问
1. **稳定性**：相比网页抓取，API更稳定可靠
2. **合规性**：通过官方渠道获取数据
3. **效率**：更高效的数据获取方式
4. **支持平台**：通过合规方式支持X内容生态

### 项目价值
1. **内容传播**：将X优质内容转化为视频，扩大传播
2. **创作者工具**：为内容创作者提供自动化工具
3. **平台生态**：丰富X内容的多平台分发

### 安全与隐私承诺
1. 仅访问公开推文数据
2. 不收集用户个人信息
3. 不进行垃圾信息发送
4. 遵守所有X平台规则

## 申请填写建议

### 使用目的
选择："Making a bot" 或 "Academic research"

### 详细描述
"开发一个自动化视频创作系统，将X上的热门讨论转化为短视频内容，帮助优质内容获得更多曝光。"

### 预期使用量
- 每月API调用：~15,000次
- 每小时请求：~1次
- 每次获取：20-50条推文

### 数据使用
"仅用于内容分析和视频创作，不存储原始推文数据，生成的内容会进行二次创作。"

## 备用方案说明

如果API申请未通过，项目已实现：
1. **模拟数据系统**：使用AI生成测试内容
2. **降级方案**：可切换其他数据源
3. **手动输入**：支持手动导入内容

但API访问将大大提升系统质量和稳定性。
