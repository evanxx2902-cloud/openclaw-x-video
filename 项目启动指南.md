## 🔧 核心功能

### 1. 智能数据抓取 (`skills/scraper/`)
- **优先**：X官方API（需要配置 `X_API_KEY`）
- **备用**：高质量模拟数据（立即可用）
- **配置**：`config/.env` 中的 `X_API_KEY`

### 2. AI内容分析 (`skills/analyst/`)
- 使用DeepSeek API分析内容价值
- 判断视频类型：Type A（文字动画）或 Type B（故事混剪）
- 生成结构化任务JSON

### 3. 串行渲染队列 (`skills/queue/`)
- 文件锁防止并发渲染
- 2C4G服务器内存保护
- 任务状态跟踪（pending/completed/failed）

### 4. 视频渲染 (`skills/remotion/`)
- **Type A**：文字动画（Remotion React）
- **Type B**：故事混剪（FFmpeg + 字幕）
- **质量配置**：fast/balanced/high_quality
- **服务器优化**：单线程渲染，避免OOM

## ⚙️ 配置说明

### 必需配置 (`config/.env`)
```env
# LLM API
LLM_API_BASE=https://api.deepseek.com
LLM_API_KEY=sk-xxxxxxxx
LLM_MODEL=deepseek-chat

# X API（可选，没有则用模拟数据）
X_API_KEY=你的_X_API_密钥

# X账号（用于网页抓取备用方案）
X_USERNAME=你的用户名
X_PASSWORD=你的密码

# TTS配置（Type B视频）
TTS_VOICE=zh-CN-XiaoxiaoNeural
TTS_RATE=+10%

# Remotion渲染
REMOTION_CONCURRENCY=1
```

### 可选配置
- **背景视频**：将MP4文件放入 `data/bg_videos/`
- **字体文件**：可添加中文字体到 `skills/remotion/public/`
- **代理设置**：如需代理访问X API

### 常见问题排查

#### 1. 视频渲染失败
```bash
# 检查Remotion依赖
cd skills/remotion
npm list --depth=0

# 检查Node.js版本
node --version  # 需要 >= 22.12

# 检查内存使用
```

#### 2. LLM API错误
```bash
# 测试API连接
python -c "
import os
from dotenv import load_dotenv
load_dotenv('config/.env')
print('API Key:', os.getenv('LLM_API_KEY')[:10] + '...')
"

# 检查网络连接
curl -I https://api.deepseek.com
```

#### 3. 队列卡住
```bash
# 检查队列状态
python -c "
import sys
sys.path.insert(0, '.')
from skills.queue.queue_manager import status
import json
print(json.dumps(status(), indent=2))
"

# 重置队列（谨慎使用）
echo '{\"current\": null, \"pending\": [], \"completed\": [], \"failed\": []}' > data/render_queue.json
```

## 🚀 生产部署建议

### 1. 服务器要求
- **最低**：2核CPU，4GB内存，20GB磁盘
- **推荐**：4核CPU，8GB内存，50GB磁盘
- **网络**：稳定访问X API和LLM API

### 2. 安全配置
```bash
# 保护配置文件
chmod 600 config/.env

# 限制日志访问
chmod 750 logs/

# 定期清理旧文件
find data/tasks/ -name "*.json" -mtime +7 -delete
find output/ -name "*.mp4" -mtime +30 -delete
find logs/ -name "*.log" -mtime +30 -delete
```

### 3. 备份策略
```bash
# 备份重要数据
tar -czf backup_$(date +%Y%m%d).tar.gz data/tasks/ output/ config/.env

# 定期备份到外部存储
# 建议每周备份一次
```
